# Screenshot Capture Checklist

Save README screenshots in this folder using the exact filenames below.

Recommended browser width: 1600px or wider if possible. Capture the main dashboard content, including the left navigation and top merchant title.

The root images are the original captures. Cropped README-ready versions live in `readme/` and are the ones linked from the project README.

## Required

1. `01-overview-dashboard.png`
   - View: Overview
   - Show balance cards, recent payment, and reconciliation summary.

2. `02-customers-invoices.png`
   - View: Invoices
   - Show at least one customer-created invoice in `PAID` or `ISSUED` status.

3. `03-payments.png`
   - View: Payments
   - Show a successful payment with provider reference, amount, fee, and net amount.

4. `04-settlement.png`
   - View: Settlement
   - Show a settlement batch with gross, refunded, and net amounts.

5. `05-reconciliation.png`
   - View: Reconciliation
   - Show a completed report with matched count and exception count.

6. `06-audit-logs.png`
   - View: Audit
   - Show audit events for payment and settlement actions.

## Optional

7. `07-login.png`
   - View: Login/Register screen
   - Useful if we want the README to show how the demo starts.

8. `08-refunds.png`
   - View: Refunds
   - Useful if you create a partial refund and want to show refund behavior.

## Notes

- Do not include real credentials, tokens, real bank details, or personal information.
- The current mock account and sample names are fine because this project is a simulation.
- Keep filenames lowercase and exactly as listed so the README links stay simple.
